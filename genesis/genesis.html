Projeto Gênesis

1. O que é e como funciona a contratação de um domínio?

O domínio é o endereço do seu site na internet (como ://seusite.com.br). Ele serve para que as pessoas não precisem digitar um número de IP complexo no navegador.Onde registrar: O registro é feito em empresas chamadas "registradoras". No Brasil, o órgão oficial para domínios .br é o Registro.br. Para domínios internacionais (como .com, .net), usam-se empresas como Namecheap, GoDaddy ou HostGator.Pagamento: O domínio não é comprado de forma definitiva. Você paga uma anuidade para ter o direito de uso daquele nome.Disponibilidade: Antes de comprar, você deve pesquisar se o nome desejado já não foi registrado por outra pessoa.

2. Preparando o Servidor LinuxPara o site funcionar

Você precisa de um computador ligado 24 horas por dia conectado à internet. Geralmente, contrata-se um servidor virtual na nuvem (chamado de VPS) em provedores como DigitalOcean, AWS, Google Cloud ou Linode.Ao criar esse servidor, escolhe-se uma distribuição Linux (sendo o Ubuntu Server ou o Debian as opções mais comuns e fáceis de usar).O acesso ao servidor é feito de forma segura e por linhas de comando através de um protocolo chamado SSH (Secure Shell).

3. Configurando o Servidor WebCom o servidor Linux acessado,

o trabalho consiste em instalar e configurar os programas que vão processar e entregar as páginas web aos visitantes. O processo padrão envolve:Instalar o Servidor Web: Os programas mais utilizados para isso são o Apache ou o Nginx. Eles interceptam os acessos vindos da internet e mostram os arquivos do seu site.Instalar o Banco de Dados e Linguagens: Se o site for dinâmico (como um WordPress), instala-se o interpretador da linguagem (geralmente PHP or Node.js) e um banco de dados (como MySQL ou MariaDB). Esse conjunto de ferramentas no Linux é conhecido pela sigla LAMP ou LEMP.Configurar as Pastas: Os arquivos do site (HTML, CSS, imagens) são colocados em uma pasta específica do Linux (geralmente em /var/www/).

4. Apontando o Domínio para o Servidor (DNS)

de ter o domínio comprado e o servidor Linux pronto, você precisa "conectar" os dois. Isso é feito através do DNS (Sistema de Nomes de Domínio).Você pegará o número de IP público do seu servidor Linux.No painel onde você comprou o domínio, você criará um Apontamento do Tipo A.Esse apontamento diz basicamente o seguinte para a internet: "Quando alguém digitar seusite.com.br, envie essa pessoa para o IP X.X.X.X".

5. Segurança (Certificado SSL)

fim, nenhum site moderno pode rodar sem segurança. É necessário instalar um certificado digital para que o site use o protocolo seguro HTTPS (o famoso cadeado verde na barra do navegador).No Linux, isso costuma ser feito de forma gratuita usando uma ferramenta chamada Certbot, que gera e renova os certificados da entidade Let's Encrypt automaticamente.

Conhecimentos: SysAdmin (Administrador de Sistemas): Focado em manter os servidores, redes e serviços funcionando corretamente. É a figura tradicional que configura servidores web, gerencia usuários, cuida de backups e prepara o ambiente para receber a aplicação de forma estável.

Registro do Domínio

Aqui tivemos uma feliz coincidência: o órgão responsável pelos registros de domínios .br é o nic.br , Ele cuida do DNS no Brasil, e dentro das diversas categorias, eles têm subdomínios por cidade, e temos uma cidade com o meu sobrenome, por sorte, a palavra andre estava disponível dentro do domínio belem.br , assim, consegui registrar o domínio: andre.belem.br

Provedor de Cloud

Os grandes players do mercado, AWS, Google Cloud e Azure oferecem segurança e estabilidade, mas podem facilmente sair de controle orçamentário, pois além de cobrar a máquina virtual, tambem incluem custos como tráfego, banco de dados e serviços específicos.

Temos opções de Virtualização mais economicas e com preço fixo indendente dos serviços ou uso da VM, por se tratar de um projeto pessoal, procurou-se um equilibrio entre preço e estabilidade.

Latência

Esse foi o fator decisivo. A LocaWeb mostrou instabilidade já na página de compra e foi descartada. Entre as duas outras opções, o Server Farm da Hostinger fica nos Estados Unidos, o que gera uma grande latência. A diferença de performance da HostGator é na ordem de 10 vezes melhor.

A intenção é utilizar esse servidor para aplicações de IoT em tempo real, uma baixa latência melhora o tempo de resposta e eficiência do sistema, portanto, a HostGator foi a opção escolhida.

Sistema Operacional

O AlmaLinux é uma distribuição Linux de nível empresarial, de código aberto, focada em estabilidade, segurança e suporte comunitário. Ele é compatível binariamente com o Red Hat Enterprise Linux (RHEL), servindo como uma alternativa gratuita, robusta e de longo prazo, ideal para servidores e produção.

Configuração do DNS

Agora que temos o servidor na nuvem com enderçeo IP Fixo, configuramos o DNS no Registro.br para redirecionar o site andre.belem.br para o endereço IP da VM da HostGator.

Apache HTTP Server

O Apache HTTP Server é um servidor web reconhecido por sua estabilidade, segurança e alto desempenho. Sendo um software de código aberto, oferece flexibilidade por meio de módulos que permitem adicionar diversas funcionalidades conforme a necessidade. É compatível com diferentes sistemas operacionais e tecnologias, facilitando a integração com aplicações web. Além disso, conta com ampla documentação e uma comunidade ativa, tornando sua implantação e manutenção mais simples. Essas características fazem do Apache uma solução confiável para hospedar sites, APIs e sistemas corporativos.

Banco de Dados MySQL

O MySQL é um sistema de gerenciamento de banco de dados relacional amplamente utilizado por sua velocidade, confiabilidade e facilidade de administração. Sendo de código aberto, oferece excelente desempenho para aplicações web e corporativas, além de suportar grandes volumes de dados com segurança. Sua compatibilidade com diversas linguagens de programação e plataformas o torna uma solução versátil e eficiente para armazenar e gerenciar informações.
